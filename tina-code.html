Part 1 -- Data Description

The datasets assigned to our group are 6 datasets with AirBnb listing information for the cities London, Rome, and Budapest, one file for each weekdays and weekends. Each dataset has 20 variables but varying numbers of observations (all the datasets combined into one has 23042 observations). The variables are as follows:

realSum (numeric): the full price of accommodation for two people and two nights in EUR
room_type (categorical): the type of the accommodation
room_shared (boolean): dummy variable for shared rooms
room_private (boolean): dummy variable for private rooms
person_capacity (numeric): the maximum number of guests that can stay in the room
host_is_superhost (boolean): dummy variable for superhost status
multi (boolean): dummy variable if the listing belongs to hosts with 2-4 offers
biz (boolean): dummy variable if the listing belongs to hosts with more than 4 offers
cleanliness_rating (numeric): cleanliness rating
guest_satisfaction_overall (numeric): overall rating of the listing
bedrooms (numeric): number of bedrooms (0 for studios)
dist (numeric): distance from city centre in km
metro_dist (numeric): distance from nearest metro station in km
attr_index (numeric): attraction index of the listing location
attr_index_norm (numeric): normalised attraction index (0-100)
rest_index (numeric): restaurant index of the listing location
attr_index_norm (numeric): normalised restaurant index (0-100)
lng (numeric): longitude of the listing location
lat (numeric): latitude of the listing location

Source: Gydi, Kristf, Nawaro, ukasz. 2021. Airbnb Prices in European Cities. Kaggle. https://www.kaggle.com/datasets/thedevastator/airbnb-prices-in-european-cities/data

Part 2 -- Question

original question: How do the variables dist (distance from city center), metro_dist (distance from nearest metro station), and attr_index (attraction index of the listing location) correlate with overall guest satisfaction in the cities London, Rome, and Budapest?

This question is focused on inference because the goal is to understand the relationship between the variables related to the location of the listing and satisfaction of the listing.

Part 4 -- Methods and Plan

After discussing with the group, we have found a common direction which is to 1. each find the best predictors for airbnb prices each with different feature selection techniques and 2. create a final predictive model with those predictors to predict prices in our combined group project.

New question of exploration: Which variables are the most statistically significant in a multiple linear regression model for predicting AirBnb prices in the cities London, Rome, and Budapest?

We will use a multiple linear regression model since the airbnb price is numeric and continuous. To decide which variables, we will first consider all variables and use stepwise selection -- StepAIC (Stepwise Akaike Information Criterion) to determine the most statistically significant ones. We made this choice because it is an dependable and efficient way to reduce a large set of variables to just a few of the most signficant ones by evaluating AIC at every step while avoiding overfitting.

To be able to use stepwise selection, we make the following assumptions we make for multiple linear regression:

  • the relationship is linear between predictors and response variables
  • independent observations
  • low multicollinearity
  • constant errors between all levels of predictors
  • residuals follow a normal distribution. In addition, there are some other assumptions specific to stepwise
  • sufficiently sized data

In addition, we make the following assumptions for stepwise selection.

  • results should be robust across different samples -- we will need to handle null values as it may cause biased variable selection

The chosen method has the following limitations:

  • if above assumptions do not hold, model's reliability decreases
  • risk of overfitting with too many preedictors
  • interactions and non linear patterns are not captured with MLR
  • Stepwise selection may remove important variables or include unimportant ones due to its local optimization nature or if any assumptions are violated
